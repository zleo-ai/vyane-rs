# WP-67 · Resident broker/projector daemon assembly

Status: in progress

## Goal

Use the existing owner-scoped message and AgentRun projection substrate from a
resident daemon without introducing a second queue, a second owner scope, or
an implicit delivery adapter.

## Delivered slice

- `VyaneService::resident_broker` assembles one `ResidentBrokerSupervisor`
  from the message broker/projector and an exact owner-matched
  `AgentProjectionComponents` instance; the CLI does not reopen those
  components itself.
- The daemon starts that supervisor on every supported platform and cancels it
  during graceful shutdown, awaiting the task before releasing the supervisor
  lock.
- The current daemon lane set is intentionally empty: maintenance plus the
  body-free message/AgentRun projectors run, while delivery adapters remain an
  explicit future integration.
- Scope mismatch is rejected before the supervisor starts; construction and
  cancellation have hermetic service coverage.

## Non-goals

- This does not add an A2A/Channels/IM adapter, remote delivery, or a second
  durable queue.
- It does not claim authenticated collaboration or whole-system daemon parity.

## Verification gate

- `cargo fmt --all -- --check`
- `cargo clippy --workspace --all-targets -- -D warnings`
- service resident-broker assembly test
- daemon workflow and AgentRun acceptance suites
- `cargo test --workspace`

The work remains a substrate assembly step toward OBS-03/INT-04/COL-03; those
parity rows must not be promoted solely because this supervisor is running.
