# WP-52 — Fresh-sessionless native authority composition

Status: delivered

Integrated into the current clean public baseline.

## Purpose

Let an admitted paired in-process AgentRun operation bind its lifetime-bound
effect authority to one exact native execution scope without exposing the raw
store or active permit and without enabling session continuity.

## Delivered contract

Private `AgentPermitState` owns the non-cloneable `ActiveExecutionPermit` and
raw `AgentStore`. Both permit-only and native-scope validation run on Tokio's
blocking pool and map store diagnostics into closed body-free errors.

`InProcessEffectAuthority::bind_fresh_native_scope` rejects any logical session
or resume proof, requires the permit policy digest to match, and performs one
immediate atomic `validate_native_execution_permit`. The returned
`InProcessNativeAuthority` is lifetime-bound to the operation authority,
non-`Clone`, non-serializable and opaque. Every positive one-based model send
or tool operation repeats the same atomic validation of owner, run, worker,
generation, controller, lease, deadline, target, prompt, policy and session
binding. Invalid coordinates fail before store access; checkpoint,
session-commit and unknown effects remain unsupported.

The standalone `AgentRunModelToolAuthority` now reuses the same private state
and effect validator without changing its public behavior. Permit-only
`InProcessEffectAuthority::authorize` remains available for non-native input or
result boundaries, but its proof is explicitly insufficient for a native loop.

## Verification

Hermetic tests cover exact scope success, target/prompt/policy drift,
session/resume rejection, expiry between effects, invalid/unassembled effects
before store validation, opaque diagnostics and static non-Clone/non-Serde
properties. The crate also passes its Rust 1.85 all-target check.

## Non-goals and remaining parity work

WP-52 provides no request resolver, prompt/policy digest construction,
authorized model client, tool registry, concrete AgentRun operation, output
sink or crash-consistent result receipt. At the WP-52 boundary, publishing
output and settling an AgentRun therefore remained separate truths. WP-53 now
adds the generic completion receipt, sink reconciliation, and publication
contract, but still does not supply a concrete product operation. Session-aware
authority, checkpoints, trusted tools, Process/Remote backends and production
host wiring remain open.
