# WP-75 · Review handback

## Outcome

Extend the explicit continuity approval boundary from the takeover step to the
immediately dependent review step. A successful takeover releases
`review_takeover`; its approval binds the exact terminal takeover approval and
run evidence. Successful review hands control back by removing the review wait
from `resume_primary`, while quota reset remains an explicit unmet dependency.

## Contract

- `continuity-queue` selects only supported approval-required ready steps:
  `takeover/start_takeover` or `review_takeover/review_takeover_work`.
- A review row binds the exact successful takeover approval ID, run ID and run
  status, in addition to the existing goal/plan/target/workdir/sandbox/timeout
  snapshot.
- Queue and decision never dispatch. Execution accepts only the durable
  approval ID and revalidates all bound evidence before consuming it.
- Review consumption and the visible `in_flight` transition are atomic;
  terminal run evidence and the visible `done`/`blocked` transition are atomic.
- Successful review changes `resume_primary` only from
  `waiting_for_quota_reset_and_review` to `waiting_for_quota_reset`. It does not
  infer quota availability, dispatch primary work, or resume the goal.

## Explicit exclusions

- quota connector, reader, runner or reset signal ingestion
- primary-resume approval or execution
- authenticated HTTP/MCP goal service
- review-check/PR adapters and repair loops
