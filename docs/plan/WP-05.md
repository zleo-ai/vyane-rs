# WP-05 — Ledger & Sessions

Maps to roadmap **M5**. Crate: `vyane-ledger`.

## Goal

Persist runs and sessions. A `JsonlLedger` appends `RunRecord`s to an
append-only JSONL file safely across processes; an `FsSessionStore` keeps one
JSON file per session with atomic writes. Cost estimation fills `cost_usd` from
a price table, and **never guesses** an unknown model.

## Scope (modules)

- `jsonl` — `JsonlLedger` implementing `vyane_core::Ledger`.
- `session` — `FsSessionStore` implementing `vyane_core::SessionStore`.
- `cost` — builtin price table + config override + estimator.

Uses `fs4` (workspace dep) for the advisory file lock; `serde_json` for line
encoding; `tempfile` (dev) for tests. No new dependency.

## Behaviour

- **`JsonlLedger` (append-only JSONL).**
  - `append`: serialize one `RunRecord` to a single line (`\n`-terminated),
    acquire an **advisory file lock** (`fs4`) around the write for cross-process
    safety, and write the whole line atomically (one `write_all` of the
    line-plus-newline; no partial-line interleaving under concurrency).
  - `query`: **reverse-scan** the file (most-recent-first) applying `RunQuery`
    filters — `owner`, `provider`, `status`, `since`, `limit`. A **corrupt line
    is skipped with a counted warning**, never a panic; the scan continues.
- **`FsSessionStore` (one JSON file per session).**
  - `save`: write the `SessionRecord` to `<dir>/<session_id>.json` via **tmp +
    atomic rename** (write a temp file in the same dir, then `rename` over the
    target) so a reader never sees a half-written file.
  - `load`: return `Ok(None)` for a missing session (not an error);
    deserialize otherwise.
  - `list`: enumerate records, filtered by `owner` when `Some`.
- **Cost estimation.**
  - A static builtin price table: per-model input/output USD per **1,000,000**
    tokens, for a few well-known models.
  - Overridable via a config table (config-provided entries win over builtins).
  - Estimate from `Usage` (input + output, cached/reasoning handled per the
    table's convention). **Unknown model ⇒ `cost_usd = None`** — never a
    guessed or zero value.

## Frozen interfaces it implements

- `#[async_trait] Ledger`: `async fn append(&self, record: &RunRecord) ->
  Result<()>`, `async fn query(&self, query: RunQuery) ->
  Result<Vec<RunRecord>>`.
- `#[async_trait] SessionStore`: `async fn load(&self, session_id: &str) ->
  Result<Option<SessionRecord>>`, `async fn save(&self, record:
  &SessionRecord) -> Result<()>`, `async fn list(&self, owner: Option<&str>) ->
  Result<Vec<SessionRecord>>`.
- Types: `RunRecord` (all fields per `run.rs`), `RunQuery { owner, provider,
  status, since, limit }`, `RunStatus`, `Usage`, `SessionRecord`; I/O failures
  as `ErrorKind::Io` (the `From<std::io::Error>` in `vyane-core` already maps
  this).

## Acceptance (mechanically checkable)

Against real temp files (`tempfile`), no external services:

- **Concurrent-append integrity:** two tasks each append 100 records to the same
  ledger concurrently; afterward the file has exactly 200 valid JSON lines, no
  loss and **no interleaved/corrupted lines** (every line parses as a
  `RunRecord`).
- **Corrupt-line tolerance:** a ledger file seeded with garbage lines between
  valid ones queries successfully, returns only the valid records, and reports a
  non-zero skipped-line count (no panic).
- **Query filters:** `owner`, `provider`, `status`, `since`, and `limit` each
  filter as specified, and results are most-recent-first (reverse-scan order).
- **Missing-session load:** `load` of an absent id returns `Ok(None)`.
- **Session round-trip:** `save` then `load` returns an equal `SessionRecord`;
  a concurrent reader during `save` never observes a partial file (atomic
  rename).
- **Cost:** a known model yields the expected `cost_usd`; a config override
  changes it; an unknown model yields `None`.

## Non-goals

- No SQL / database backend — plain files by design (see "boring dependencies").
- No ledger compaction, rotation, or retention policy in v0.1.
- No pruning or GC of session files.
- No prompt bodies: the ledger stores `RunRecord` as-is (digest + optional
  short preview); it does not archive prompt text on its own.

## Notes

- The advisory lock guards cross-process appends; within a process, keep the
  critical section tight (lock → append line → unlock).
- Atomic writes rely on same-directory `rename`; create temp files beside the
  target, not in a global temp dir, so `rename` stays atomic on the same
  filesystem.
- Treat the price table's units as fixed at **USD per 1M tokens** to avoid
  order-of-magnitude mistakes; document the convention beside the table.
