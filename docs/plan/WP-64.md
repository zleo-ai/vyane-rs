# WP-64 — Shared multi-lane resident AgentRun host substrate

## Outcome

`vyane-service` can construct a bounded resident host with one exact execution
loop per durable backend, all sharing one owner and store. Each lane freezes
its executor kind, backend, lease identity, and execution bounds before the
host starts. Backend and lease identities are unique.

The host starts exactly one recovery loop over the validated union of
controller adapters and exactly one completion projector. The adapter union may
also retain a controller kind whose execution lane is currently disabled, so
historical running records can still be reconciled. Every active lane must have
a matching adapter.

## Isolation and shutdown

Lane polling, panic accounting, and exponential backoff are independent. One
degraded lane cannot prevent another exact backend from claiming work for a
different worker. Store-level ordering remains authoritative: a later run for
one worker cannot pass an earlier queued run merely because their backends
differ.

Host cancellation stops new execution claims and cooperatively drains every
active lane. Only after all lanes return does the host stop and drain recovery;
completion remains live until both phases finish, then performs one final
bounded outbox pass. Work beyond that final batch remains durable for a later
host start. Exit statistics contain only bounded counters and the closed
backend discriminator.

## Construction boundary

Construction validates lane bounds, metadata, execution options, the complete
recovery adapter union, matching coverage, completion sinks, duplicate
backends, and duplicate lease identities without probing or mutating the
store. The existing single-lane `ResidentAgentSupervisor` and
`ResidentInProcessAgentSupervisor` APIs remain available.

This package is service-layer host substrate only. The daemon still assembles
only its existing Linux CLI-harness Process lane. No native or Remote production
executor, factory, trusted built-in set, OS sandbox, session-aware resume, or
restart replay is added or claimed here.
