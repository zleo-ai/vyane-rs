# WP-79 · Read-only continuity next-action projection

## Outcome

Project one stable, machine-readable operator next action from the current goal
continuity state and approval evidence. The projection is read-only: asking what
comes next never queues, decides, consumes, dispatches, retries, or records a
signal.

## Contract

- `vyane goal continuity-next <goal-id>` reads the exact owner-scoped goal and
  its approvals, then emits one closed action kind with the current goal
  revision and quota boundary.
- Actions distinguish queue, decide, execute, external signal, internal wait,
  in-flight execution, blocked execution, manual decision, and a completed
  continuity plan. A completed plan does not complete the goal or satisfy its
  acceptance criteria.
- Approval actions include the exact durable approval ID when one exists.
  Queue/decision/signal actions list the additional operator inputs still
  required instead of inventing executable values.
- External waits expose only the closed signal kinds accepted by the current
  state. The projection does not poll a network or infer that a signal passed.
- Projection fails closed for cross-owner/cross-goal approval input or an
  in-flight/blocked step whose matching durable approval is missing.
- Repeating the command against unchanged durable state produces the same
  action and performs no writes.

## Explicit exclusions

- no automatic queueing, approval, execution, retry or repair loop
- no GitHub/network watcher or signal ingestion
- no authenticated HTTP/MCP goal service
- no runtime-native session resume
- no goal completion or acceptance mutation

CON-07 remains `partial` after this package. The projection is the read contract
that a later optional watcher/runner may consume under separately authenticated
authority.
