# WP-42 — Bounded AgentRun integration adapters

## Goal

Add two narrow consumers of the AgentRun substrate without claiming a resident
worker runtime or a native harness: a concrete authority bridge for the only
currently safe execution scope, and a one-shot lifecycle outbox projector.

Integrated into the current clean public baseline, including the
explicit projection assembly follow-on.

## Fresh-sessionless model/tool authority

`vyane-service::AgentRunModelToolAuthority` owns one non-serializable
`ActiveExecutionPermit`, its frozen `NativeExecutionScope`, and an
`Arc<dyn AgentStore>`. Construction is deliberately limited to a fresh scope
with no logical session or resume proof. A mismatched policy digest also fails
before the value can be used.

The bridge implements `NativeExecutionAuthority` for two effects only:

- a one-based `ModelSend { turn, wire_attempt }`;
- a one-based `ToolOperation { turn, ordinal }`.

Every supported call moves the synchronous store revalidation to Tokio's
blocking pool. Zero coordinates are invalid, and checkpoint prepare/publish,
session commit, session-bearing scopes, unknown future effects, store drift,
expiry and revocation all fail closed. Debug and mapped errors do not expose
the permit, run identity, session material, policy digest or store path.

This is a concrete bridge, but not production execution assembly. No factory,
service operation, CLI, daemon, runtime or native model loop constructs it.
It does not own `SessionExecutionLease`, revalidate `NativeSessionDomain`,
authorize checkpoints/session commits, or make the ordinary unguarded chat and
tool compatibility methods safe for native execution.

## AgentRun lifecycle projection

`vyane-broker::AgentEventProjector::project_once(limit)` reads at most one
bounded, owner-scoped page from one projector's transactional AgentRun outbox.
For each item it durably appends a body-free EventLog lifecycle event using the
source event id unchanged, then marks that exact source event projected. A
failed append leaves the source row unacknowledged. A crash after append but
before acknowledgement can append the same stable id again, so EventLog
consumers must deduplicate by event id.

The projection includes only bounded worker/run identity, revisions,
lifecycle/state, event kind and source sequence. It excludes prompt and target
data, policy digests, logical/native sessions, task/trace identifiers, message
bodies, credentials and raw errors. Owner scope and projection progress are
isolated per projector.

This API performs one bounded pass and returns a `ProjectionReport`; it starts
no polling task and owns no resident lifecycle, cancellation or drain policy.

`vyane-service::AgentProjectionComponents::open` is the explicit owner-bound
assembly path. It validates the owner before creating persistent state, opens
the AgentRun database through `StoragePaths::agent_metadata_db_path`, binds the
projector to the shared EventLog directory, encapsulates the raw `AgentStore`,
and exposes only the projector. Ordinary dispatch does not call this path,
open the AgentRun database, poll the outbox, or start background work. A daemon
or embedding application must opt in and own the projector lifecycle.

## Acceptance

- Authority tests cover live permit expiry, unsupported effects, zero
  coordinates, policy/session rejection, opaque debug output, static error
  mapping and non-blocking async executor behavior while SQLite validates on
  the blocking pool.
- Projector tests cover page bounds, owner and projector isolation, the exact
  body-free payload allowlist, failed-append retry, and restart after append
  before acknowledgement with duplicate stable event identity.
- Service assembly tests reject an invalid owner before database creation and
  prove one constructed component projects only its fixed owner's events while
  leaving another owner's progress untouched.
- Repository format, Clippy-with-warnings-denied, workspace tests, independent
  review and staged secret scanning remain merge requirements.

## Open integration work

- assemble a session-aware authority that owns the live session lease and
  revalidates the exact native-session domain through checkpoint and revision-
  fenced commit;
- assemble the bounded model-to-tool-to-model driver described in
  [WP-43](WP-43.md) with this fresh-sessionless authority bridge; trusted
  built-in tools must validate actual arguments and revalidate internal
  open/publish/spawn operations independently;
- production-assemble the delivery/projection driver described in
  [WP-44](WP-44.md), and separately add an AgentRun execution/recovery
  supervisor that composes session authority and controller fencing;
- wire controller/message handback and authenticated public service, CLI, A2A
  and Channels surfaces.

Until those gates are executable, `NativeHarness`, native resume and resident
AgentRun execution remain disabled.
