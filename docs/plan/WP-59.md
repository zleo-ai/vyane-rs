# WP-59 — Owner-scoped local A2A message CLI

Status: delivered in this topic branch pending integration review.

## Purpose

Expose the existing transactional message store as a small, durable local
agent-to-agent inbox. The command surface is intended for same-machine scripts
and agent harnesses that need delayed delivery, stable JSON, explicit owner
scope, and read acknowledgement without inventing a second message database.

## Delivered contract

- `vyane-message` adds a bounded mailbox page over one exact
  `owner + route + endpoint` tuple. Default pages include live pending, leased,
  and delivered rows; acknowledged and not-yet-due rows are separately opt-in.
  Expired, cancelled, and dead-lettered rows are never inbox results.
- Exact message claim requires the same owner and mailbox tuple and reuses the
  existing lease generation, receipt fencing, attempt accounting, TTL,
  publication gate, and reclaim behavior. Unauthorized scope is
  indistinguishable from absence.
- `vyane a2a send`, `inbox`, and `read` use the standard message database or an
  explicit `--db`. `--owner` defaults to `local` and accepts
  `--owner-user-id` as a compatibility alias. JSON output has a fixed public
  field set and never includes a raw lease receipt.
- `read` requires both recipient mailbox and message id. It performs exact
  claim, delivered transition, and acknowledgement through the existing
  fenced state machine. A response loss can be reclaimed after lease expiry;
  an already-read, absent, cross-owner, cross-mailbox, or currently leased
  message produces the same safe non-success result.
- Inbox pages are non-mutating, stably ordered, bounded by 1..=1000 rows, and
  report `has_more`. Delayed messages are hidden unless `--include-future` is
  set; acknowledged messages are hidden unless `--include-read` is set.

## Verification and limits

Store contract tests cover owner and mailbox isolation, delayed visibility,
stable ordering, pagination, exact claim, and acknowledged-history opt-in. CLI
acceptance tests cover send/inbox/read, sender and unrelated mailbox isolation,
cross-owner and cross-mailbox read denial, delayed pages, stable JSON fields,
invalid payloads, invalid bounds, and explicit repeated-read failure.

Mutation proof temporarily removed the exact-claim `target_id` predicate. The
`exact_claim_fails_closed_and_acknowledged_messages_are_opt_in` contract test
then failed on its cross-mailbox denial assertion. Restoring the predicate made
that exact test and the CLI acceptance suite pass again; the mutation is not
part of the branch.

This is a local queue surface, not authenticated multi-user authority. A caller
that can invoke the process can also choose `--owner` and `--db`; deployments
must not treat either flag as principal-derived authorization. This package
does not implement A2A HTTP v0.3, Agent Cards, remote send/get/cancel,
send-subscribe, SSE, push delivery, discovery, SSRF defenses, or a Channels
adapter. Those remain separate optional-adapter work.
