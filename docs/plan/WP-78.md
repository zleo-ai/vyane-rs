# WP-78 · Review-check gating and approved repair handback

## Outcome

Extend typed goal continuity with explicit review-check pass/fail signals and
an approval-gated repair step. A failed check cannot be erased by a later pass:
the primary continuation remains blocked until the exact repair execution
finishes successfully.

## Contract

- `wait_for_review_checks_before_resume` is opt-in and requires both a reviewer
  and `require_review_before_resume`.
- The handoff plan adds a non-executable `wait_review_checks` gate and an
  approval-required `repair_failed_review` step. Primary resume depends on the
  completed wait gate instead of directly on review.
- `review-checks-passed` and `review-checks-failed` are typed, idempotent
  external signals bound to the current quota boundary, an exact public
  repository/pull-request coordinate, and a stable observation ID. Recording a
  signal never dispatches; later observations for the same PR form a bounded,
  durable sequence.
- A pass releases the wait gate only when review is done and no unresolved
  failure exists. Once a failure is observed, a later pass still waits for a
  successful repair run.
- If checks fail again after a successful repair, the latest observation closes
  the gate again, resets the repair step, and requires a new repair plus a newer
  passing observation before resume can become ready.
- Repair approval binds the exact successful review run and recursively
  revalidates review-to-takeover evidence at consumption time.
- After a failed check, primary-resume approval binds the exact successful
  repair run and recursively revalidates repair-to-review-to-takeover evidence.
- Repair and resume continue to use explicit decisions and one-shot fresh
  dispatches. Neither completes the goal nor satisfies acceptance criteria.

## Explicit exclusions

- no GitHub/network polling or automatic signal ingestion
- no automatic approval, queueing, dispatch, retry or repair loop
- no authenticated HTTP/MCP goal service
- no runtime-native session resume
- no operator next-action/command projection
