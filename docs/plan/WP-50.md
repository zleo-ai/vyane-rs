# WP-50 — Owner-qualified durable task truth, phase B

Status: delivered

Integrated into the current clean public baseline, including
artifact-namespace hardening.

## Purpose

Remove globally unique task ids from the SQLite source of truth so two durable
owners may use the same task id without sharing snapshots, events, leases, CAS
state, or output artifacts. This is a storage phase, not multi-user REST auth.

## Delivered contract

Schema v2 changes the task primary key to `(owner, id)` and gives task events
the matching composite foreign key and uniqueness constraint. Every
`TaskStore` create/read/list/event/mutation method requires owner separately;
`NewTask` no longer accepts owner as payload data. Foreign-owner mutation is
absent-shaped and cannot change the target owner's snapshot or event stream.

The v1-to-v2 migration runs inside the store's immediate transaction after an
exact v1 schema check. It preserves task/event counts, event sequence values
and the autoincrement high-water mark, checks composite foreign keys and
orphans, validates the complete v2 manifest, and rolls back on any mismatch.

REST result artifacts now use domain-separated SHA-256 owner and task path
segments. New writes never place raw owner or task ids in a path. A read-only
legacy fallback is allowed only for the existing local REST scope and a
UUID-parseable id after the durable row has been proven to be a
successful local REST task.

## Verification

Contract tests cover same-id owners across snapshots, events, CAS, leases,
pagination and recovery filters; foreign mutations; composite FK rejection;
rich v1 migration, rollback and sequence preservation; schema corruption; and
owner/task-qualified artifact paths including hostile path-shaped inputs.

## Non-goals and remaining parity work

Built-in REST, CLI detached tasks and workflow daemon still select the explicit
single-user `local` owner. The older filesystem detached-task scaffold remains
a local-only compatibility subsystem rather than a multi-owner store. REST has
not yet authenticated distinct principals into `OwnerContextFactory`, and
workflow/message/AgentRun/event control is not fully protocol-scoped. WP-50
therefore advances GOV-01 but does not close it or claim hostile same-UID
multi-user isolation.
