# WP-66 · Native AgentRun production assembler

状态：in progress（实现 worktree：`codex/native-agent-assembler`）

## 目标

把 WP-65 的 fresh/sessionless native operation 从测试专用 seam 组合成一个
明确受限的 resident AgentRun lane。生产 assembler 必须复用已有的
`AgentRunExecutionDriver`、`AgentRunRecoveryDriver`、completion projector、
`NativeAgentInputSpool`、`NativeTurnDriver` 和 authorized OpenAI Chat client；
不得绕过 durable claim、permit、effect authority 或 completion publication。

## 公开范围

- 仅支持 owner-scoped、fresh/sessionless、单 worker 的 native run。
- 仅支持已经解析且冻结的单一 OpenAI Chat target；配置、prompt、workdir
  identity、timeout 和 turn limit 在创建 durable run 前完成绑定。
- native lane 与现有 CLI-harness Process lane 共用 recovery union 和 completion
  projector，但各自拥有唯一 `ExecutionBackend`、spool 和 lease owner。
- daemon 的 API 只能返回 allowlisted lifecycle/output view；prompt、system、
  auth material、raw target endpoint 和 spool body 不得进入 durable run row 或
  public response。

## 明确非目标

- 不启用 native session resume、checkpoint/session commit、approval resume、
  automatic restart replay 或 live pause/resume。
- 不提供 trusted built-in tools、OS sandbox、Remote controller、multi-user
  principal isolation 或跨机 exactly-once。
- 不把 `NativeTurnDriver` 或一个 native lane 的存在描述为完整 `Harness` 或
  广义 multi-backend product host。

## 当前实现切片

- daemon 已通过 `execution_backend = "native_in_process"` 接受显式 native
  submit；默认值仍是兼容既有调用方的 `cli_harness_process`。
- native submit 只接受 `read_only`、显式可 pin workdir、单一 direct
  OpenAI Chat target，并在 durable run 创建前冻结 owner/run/worker、prompt、
  target、routing digest、workdir identity、timeout 与 system policy。
- daemon 以一个 `ResidentAgentHost` 同时运行 Process 与 Native lane；startup
  recovery 使用同一 adapter union，completion 只配置一个 message sink。
- hermetic daemon acceptance 已覆盖真实 HTTP submit → native wire → durable
  success → completion publication；其余 cancel/restart/shutdown race 仍按下面
  验收门继续补齐。

## 验收门

1. assembler 构造失败时不写 store、spool、controller 或 message state；lane
   的 backend/kind/lease owner 映射必须唯一且 recovery adapter 完整。
2. native submit 在 durable run 创建前完成 target/policy/workdir authority
   binding；重复 idempotency submit 只接受 exact frozen input。
3. native execution 每次 model wire send 经过 authorized client 和 live effect
   authority；target、prompt、policy、timeout、workdir 或 owner drift 在 wire
   前 fail closed。
4. cancel、controller loss、timeout、completion staging/publication 和 graceful
   shutdown 分别有 hermetic acceptance；任何 uncertain cleanup 都不得删除
   mismatched replacement spool。
5. Process 与 native lane 并发运行时互不阻塞，recovery 只 claim 对应 backend，
   completion projector 仍只发布 exact owner/run/generation binding。
6. 运行 `cargo fmt --all -- --check`、
   `cargo clippy --workspace --all-targets -- -D warnings`、`cargo test --workspace`，
   并对关键 authority guard 做至少一条 mutation test。

## 收口条件

只有在 production daemon assembly、native submit/status/output/cancel 的
allowlist、recovery/shutdown acceptance、mutation evidence 和独立评审全部完成
后，才能把 parity 的 EXE-07/EXE-09/INT-04 相关结论从当前 `partial`/`different`
向前移动；本工作包本身不宣称 whole-system parity。
