# WP-58 — Exact-plan workflow replay/fork

## Goal

Add the explicit new-run continuation mode accepted by ADR 0002. A replay
loads one terminal source journal, verifies its exact source and plan digests,
copies a dependency-closed, journal-recorded all-success prefix into a new UUIDv7 journal, and
executes the remaining steps live.

## Contract

- `resume` keeps the source run identity and journal. `replay` always creates a
  different identity and never mutates the source journal.
- Replay requires an existing plan digest and an exact source/plan match. It
  does not use the legacy digest migration allowed by compatibility resume.
- Reuse stops at the first topological step that is not a clean success, has
  any failed fan-out output, or whose dependencies are not already reused.
  Later successes are not copied.
- The new journal is published create-only before live execution. Optional
  caller-supplied identities retain the same atomic no-overwrite contract.
- Replay provenance contains only the source run id, plan digest, reused step
  ids, and a digest of the copied step evidence. It adds no prompt or output
  body beyond the ordinary local journal.
- A running source is not replayable. Daemon restart still does not imply
  resume or replay.

## Product surface

`vyane workflow replay SOURCE_ID --file WORKFLOW.toml [--id UUIDV7] [--json]`
loads variables from the source journal. `--var` is rejected. Before target
journal publication or live suffix execution, the CLI flushes the exact new id
to stderr so an interrupted caller can reconcile instead of retrying blindly.

## Non-goals

- changed-plan or fuzzy call-hash matching;
- dynamic control flow, nested workflows, or shared budget;
- daemon automatic replay;
- REST/MCP raw plan transport;
- a cross-implementation parity claim.

## Acceptance

- new identity and byte-for-byte unchanged source journal;
- exact digest and source-shape checks before resolver or dispatch effects;
- partial replay reuses only the successful prefix and executes the rest once;
- completed replay performs no repeated model calls;
- create-only target publication and public-repository hygiene;
- workspace format, Clippy, tests, and independent review gates.
