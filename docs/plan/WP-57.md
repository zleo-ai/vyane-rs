# WP-57 — Sanitized automatic-routing parity suite

Status: delivered in this topic branch pending integration review.

## Purpose

Make automatic-routing parity falsifiable without publishing private repository
provenance, deployment identities, prompts, endpoints, or credentials. This
work extends the public behavior manifest; it does not turn a selected fixture
set into a claim of whole-system parity.

## Delivered contract

- The versioned manifest adds ten neutral automatic-routing cases for routed
  defaults, configured and typed workflow effort precedence, explicit selector
  bypass, actual template rendering, no-eligible failure, full-chain frontier
  filtering, ambiguous direct failover rejection, and frozen route replay.
  Only reference-native profile effort and no-eligible behavior are counted as
  normalized matches; Rust-specific target mapping remains open.
- Each case pins the sanitized reference input/output digest, public fixture
  SHA-256, case set, disposition, and current Rust output. The Rust test
  recomputes the actual target chain, canonical routing labels, effective
  effort, eligibility failure, and replay result using only local fixtures.
- Automatic explicit-model override remains an honest open difference. Default
  and rendered-task target mapping, explicit selector resolution, explicit
  effort plus Rust target mapping, profile-chain frontier filtering, direct-leg
  ambiguity, and frozen automatic target-chain replay are also explicitly open.
  None is counted as reference equality.
- `.github/scripts/parity-report.py` runs the Rust parity test by default,
  validates every stored fixture and disposition offline, and emits stable JSON
  or Markdown. It validates a stored sanitized attestation; it does not execute
  the private reference. `--skip-rust-test` is an explicit stored-only mode.

## Verification and limits

The suite requires no network or model credential and uses only reserved
example domains and neutral identities. Exact/normalized cases cannot retain a
blocker; an open difference must remain unequal and carry a public `BLOCKER`.
Unknown fields and nested schema drift fail closed in the Rust test; the stored-
only report closes top-level keys, case sets, digests, duplicate identifiers,
paths, and dispositions. The default report runs both layers.

This slice does not cover target-schema migration, production attempt trails,
run/session schemas, stateful routing signals, or a production-complete model
tier policy. QUA-04 and EXE-05 therefore remain partial/different respectively.
