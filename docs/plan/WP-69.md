# WP-69 · Durable goal verification artifacts

Status: in progress

## Goal

Preserve every bounded acceptance-verification attempt as immutable,
owner-scoped evidence without turning that evidence into completion or pursuit
authority.

## Delivered slice

- schema v3 adds append-only `goal_verifications` rows keyed by a UUIDv7
  verification identity and `(owner, goal_id)` foreign key.
- Each artifact stores the complete bounded `AcceptanceVerification` JSON, its
  SHA-256 digest, verification instant, and optional lease worker identity.
- Writes require an in-progress goal and reuse the existing owner/active-lease
  fence. Reads use foreign-as-absent owner semantics.
- `vyane goal verify` records the artifact before attempting criterion
  mutations, so a later fenced persistence error cannot erase already-computed
  evidence. `vyane goal get --json` returns the immutable attempt history.
- Artifact payloads are bounded to 1 MiB and reads return the latest 100 in
  chronological order. The SQLite database remains private and verification
  rows reject update/delete at the database boundary. Retention/GC remains a
  later policy rather than making long-lived goals permanently unverifiable.

## Non-goals

- Artifacts do not auto-complete goals, authorize pursuit, waive criteria, or
  grant approval/kernel takeover.
- No network, GitHub, board, quota, or authenticated service adapter is added.
- The artifact digest detects stored corruption; it is not a signature or
  external provenance claim.

## Verification gate

- `cargo fmt --all -- --check`
- `cargo clippy --workspace --all-targets -- -D warnings`
- `cargo test --workspace`
- schema migration, owner isolation, immutability, digest, and CLI acceptance
  tests

CON-07 remains `partial`: WP-70 adds explicit manual pursuit; automatic/resumable
pursuit, quota/continuity handoff, approval/kernel takeover, and authenticated
service authority are still open.
