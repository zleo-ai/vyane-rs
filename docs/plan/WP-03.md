# WP-03 — Harnesses

Maps to roadmap **M3**. Crate: `vyane-harness`.

## Goal

Wrap coding CLIs as execution shells. Implement `vyane_core::Harness` for Claude
Code and Codex CLI: headless one-shot invocation, a child environment built
**exclusively** via `EnvPolicy::build`, and process-group spawn + group kill so
a cancel or timeout leaves no orphaned grandchildren.

## Scope (modules)

- `claude_code` — `Harness` for Claude Code.
- `codex_cli` — `Harness` for Codex CLI.
- `spawn` — shared process-group spawn + group-kill (`setsid`, kill the group),
  timeout enforcement, and env materialization from `EnvPolicy`.
- `parse` — extract the final answer text, native session id, and usage from
  each CLI's machine-readable output.

Process control via `tokio` (workspace features include `process`, `signal`,
`time`). No new dependency.

## Behaviour

- **Argv construction (per harness).** Build the headless one-shot command:
  select the model, map `job.sandbox` (`Sandbox::{ReadOnly, Write, Full}`) to
  the CLI's permission flags, set the machine-readable output format for
  parsing, and add the resume flag from `job.resume` when continuing a native
  session.
- **Child environment — via `EnvPolicy` only.** The child env is built
  **exclusively** through `EnvPolicy::build(parent_snapshot)`. The policy's
  `inject` set carries the endpoint / key / model as the env vars each CLI
  expects (`job.endpoint` may be `None`, meaning the harness authenticates
  natively — inject nothing for auth). Never pass the parent environment
  through by hand; that is the entire point of the scrub.
- **Process group.** Spawn the child in its own process group (`setsid`). On
  cancel (`CancellationToken`) or `job.timeout`, **kill the whole group**, not
  just the direct child — coding CLIs fork helper subprocesses and a bare child
  kill leaves them running.
- **Output parsing.** From the CLI's machine-readable output, produce
  `HarnessOutcome { text, native_session_id, usage, exit_code, duration }`.
  `native_session_id` is the CLI's own session id for later resume; `usage` is
  populated only when the CLI reports it.
- **`available()`.** Probe binary presence/executability (does not run a task).
- **Error classification** onto `vyane_core::ErrorKind`:
  - binary missing / not executable → `SpawnFailed`
  - ran but exited non-zero → `HarnessFailed`
  - `job.timeout` elapsed → `Timeout`
  - cancelled via token → `Cancelled`

## Frozen interfaces it implements

- `#[async_trait] Harness`: `fn kind(&self) -> HarnessKind`,
  `async fn available(&self) -> bool`,
  `async fn run(&self, job: HarnessJob, cancel: CancellationToken) ->
  Result<HarnessOutcome>`.
- Consumes `HarnessJob { prompt, model: ModelId, endpoint: Option<Endpoint>,
  params: GenParams, workdir: Option<PathBuf>, sandbox: Sandbox, resume:
  Option<String>, env: EnvPolicy, timeout: Option<Duration> }`.
- Produces `HarnessOutcome { text, native_session_id: Option<String>, usage:
  Option<Usage>, exit_code: i32, duration: Duration }`.
- `HarnessKind::{ClaudeCode, CodexCli}`; env via `EnvPolicy` / `BASELINE_ENV`.

## Acceptance (mechanically checkable)

CI uses **fake CLI shell scripts**, never real coding CLIs:

- **Argv correctness:** a fake CLI that echoes its argv proves model, sandbox
  mapping, output-format, and resume flag are constructed as specified for each
  harness.
- **Answer + session + usage parsing:** a fake CLI emitting canned
  machine-readable output yields the expected `text`, `native_session_id`, and
  `usage`.
- **Kill-tree test:** the fake CLI spawns a long-lived grandchild that writes a
  liveness file; after cancel/timeout, assert the grandchild is dead (the
  liveness file stops updating / the pid is gone). Proves the *group* is killed.
- **Env-scrub test:** set a parent `*_API_KEY`; a fake CLI dumps its own
  environment; assert the parent key is **absent** from the child and only the
  `EnvPolicy.inject` values are present. Proves parent credentials never reach
  the child.
- **`available()`:** returns `false` for a missing binary, `true` for a present
  fake CLI on `PATH`.
- **Error kinds:** missing binary → `SpawnFailed`; non-zero exit →
  `HarnessFailed`; elapsed timeout → `Timeout`; token cancel → `Cancelled`.
- **Real-CLI smoke tests** exist but are `#[ignore]` so CI never depends on a
  real coding CLI being installed.

## Non-goals

- No config parsing (WP-01 builds the `HarnessJob` and its `EnvPolicy`).
- No HTTP protocol behaviour (WP-02).
- No cross-target failover (kernel, WP-04).
- No streaming of harness output (v0.1 harnesses are one-shot; final text only).

## Notes

- `EnvPolicy::build` is a pure function; snapshot the parent env once and pass
  it in, so a run's environment is reproducible.
- Group kill must be robust to a child that ignores `SIGTERM` — escalate to
  `SIGKILL` on the group after a short grace period.
- Keep prompt text out of logs; log argv and env *keys* only, never injected
  values.
