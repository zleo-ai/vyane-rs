# WP-61 — Authenticated Process AgentRun host

## Outcome

The Linux workflow daemon also hosts a bounded, authenticated AgentRun control
surface for fresh, sessionless CLI-harness work. A submitted request is frozen
before the durable queue row is created, executed by the resident AgentRun
supervisor, and handed back through the transactional message store. This is a
production assembly for the narrow `Process` path; it is not a native-agent or
remote-agent host and does not establish whole-system parity.

## Control surface

The daemon's existing loopback listener and per-start bearer protect four
additional routes:

- `POST /v1/agent-runs` submits a caller-generated UUIDv7, task, target,
  sandbox, optional workdir/system text/timeout, and bounded labels.
- `GET /v1/agent-runs/:id` returns an allowlisted lifecycle view.
- `GET /v1/agent-runs/:id/output` returns the body only after exact completion
  publication for a succeeded run.
- `POST /v1/agent-runs/:id/cancel` requests durable tree cancellation and
  stops an exact verified process group when one exists.

The request schema denies unknown fields and exposes no owner, session,
controller, policy digest, target snapshot, capability snapshot, or arbitrary
config injection. The daemon binds all runs to its fixed local single-user
scope. The bearer is not a distinct principal and is not a hostile same-UID or
multi-user security boundary.

## Frozen input and authority

Before queue creation the host prepares the harness dispatch once and writes an
owner-private, create-only input spool. The spool binds the prompt digest to the
resolved target chain, capability plan, canonical workdir, sandbox, system
text, timeout, and labels. It accepts only fresh, sessionless CLI harness
targets. HTTP protocol targets, native execution, logical sessions, resume, and
caller-supplied serialized config remain closed.

The executor re-reads and validates that frozen input under an active permit,
recomputes the prepared dispatch, and requires exact target and capability
snapshots before any process effect. The live authority callback revalidates
owner, run, worker, generation, lease, target, prompt, and policy immediately
before wrapper spawn and each real-target release. A retry with the same UUIDv7
is idempotent only when it matches the exact durable input; a conflicting retry
does not replace or replay the original payload.

## Process lifecycle and recovery

On Linux, the host durably reserves a private controller sidecar before the
AgentRun store may publish `Running`. That never-started reservation is advanced
to the exact process identity and birth fingerprint before a real target is
released. Recovery can therefore prove a reserved controller gone without
signalling a PID; for a started controller it revalidates the identity before
TERM, before KILL, and before declaring the controller gone. PID or
process-group reuse fails closed. Sequential failover records the old controller
stopped before admitting the next attempt. Each sidecar is also bound to the
run, worker, and generation that reserved it. After startup recovery, terminal
database truth may retire a matching `Reserved` or provably empty `Stopped`
sidecar left by a crash between settlement and unlink; this bounded sweep never
signals a process and never deletes `Started` evidence.

Success requires both a typed harness success and a quiesced process lifecycle.
An exit code of zero without the Claude JSON terminal `result` field is a
redacted harness failure, including when the field is absent or has the wrong
type. The output body is staged under a stable completion key and becomes
readable only after the durable AgentRun completion and message publication
agree. AgentRun rows and completion receipts remain body-free.

At startup the daemon performs an exact stale-controller recovery pass before
accepting work, then runs bounded execution, recovery, and completion loops.
Restart never replays an input. Graceful shutdown closes AgentRun admission,
drains already admitted submit/cancel initializers, then signals and awaits the
resident AgentRun supervisor concurrently with workflow-supervisor drain. The
AgentRun supervisor forwards cancellation into an already-polled executor; the
Process harness terminates and reaps the exact group, persists the stopped
lifecycle, and completion publication gets a final drain. This is cooperative
no-orphan shutdown, not permission for an active task to run naturally to
completion. Abrupt process or machine termination still relies on the next
exact recovery pass.

## Acceptance

Hermetic Linux acceptance uses a fake Claude CLI to cover:

- successful submit, status, completion publication, output, and exact retry;
- exit zero without terminal proof failing rather than becoming success;
- exact process-group cancellation, including a child process;
- daemon restart cleanup with no automatic payload replay;
- admission closure and graceful supervisor drain, including a TERM-ignoring
  parent and child that must both be reaped without an orphan.

Unit and service tests cover private spool integrity, frozen snapshot drift,
permit fencing, sequential controller transitions, exact sidecar recovery, and
crash-consistent completion publication. Workspace fmt, clippy, and tests
remain required gates.

## Non-goals

- No `Remote` controller or A2A/Channels transport.
- No native production harness, trusted built-in tools, or host-enforced OS
  sandbox.
- No logical session, checkpoint, approval, or native resume.
- No live pause/resume or automatic restart replay.
- No distinct authenticated principals, hostile same-UID isolation, or
  multi-user daemon exposure.
- No claim of full parity with the broader reference system.
