# WP-51 — Resident paired in-process AgentRun supervisor

Status: delivered

Integrated into the current clean public baseline.

## Purpose

Compose the paired one-shot execution and recovery seams into an explicit
resident library driver without inventing a product operation, second queue,
automatic replay policy, or protocol host.

## Delivered contract

`InProcessAgentComponents::into_resident_supervisor` consumes the exact
owner/store/backend pairing into a non-`Clone`
`ResidentInProcessAgentSupervisor`. Its consuming `run` future polls execution
and recovery in independent loops. Every pass is still a bounded existing
driver; a loop never has more than one pass of its kind in flight.

Poll intervals and initial/maximum error backoff are validated, non-zero and
capped at 24 hours. Empty batches wait, full healthy batches yield, and any
degraded item, returned error or caught panic enters capped exponential
backoff. Exit reports contain only saturating body-free counters.

The host cancellation token stops new passes and interrupts scheduling waits.
It is deliberately not passed into an already-started execution or recovery
pass: each current pass receives an independent token and drains before the
supervisor returns. This separates host shutdown from AgentRun cancellation.
Dropping the outer future forfeits graceful drain, and a custom blocking store
means no fixed wall-clock drain bound can be promised.

## Verification

Hermetic tests prove the surface is non-`Clone`, invalid scheduling performs no
claim, host cancellation waits for an active operation to quiesce and settle,
and uncertain execution converges through exact-controller recovery. A
resume-eligible interruption remains interrupted: the supervisor never calls
the durable resume enqueue primitive.

## Non-goals and remaining parity work

This driver starts no task, channel, runtime, payload queue or second truth
source. It has no concrete product operation, Process/Remote backend,
tree-cancellation consumer, message input/result handback, public service/CLI/
REST surface, production host assembly, session-aware resume, live pause/resume
or automatic replay. It therefore advances CON-06/COL-03/INT-04 evidence but
does not close those rows or change their status.
